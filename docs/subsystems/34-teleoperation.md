# 遥操作系统

## §0 — One-liner

遥操作系统通过人类操作者远程控制机器人执行精细操作任务，是获取高质量专家演示数据的核心手段之一。

## §1 — 核心概念

### 1.1 遥操作的定义与分类

遥操作 (Teleoperation) 指**人类操作者通过接口远程控制机器人**执行任务的技术。

| 分类维度 | 类型 | 说明 |
|----------|------|------|
| 控制方式 | 直接遥操作 | 操作者直接控制关节/末端执行器 |
| | 监督遥操作 | 操作者设定目标，机器人自主执行 |
| | 共享控制 | 人机共同控制，各负责部分自由度 |
| 反馈方式 | 单工 | 仅操作→机器人 |
| | 双工 | 操作↔机器人（力反馈） |
| 空间关系 | 同地 | 操作者与机器人在同一空间 |
| | 异地 | 操作者与机器人分离（网络延迟） |
| 映射方式 | 关节空间 | 主设备关节→从设备关节 |
| | 任务空间 | 主设备位姿→从设备位姿 |

### 1.2 遥操作在机器人学习中的价值

1. **专家数据获取**：人类操作者提供高质量演示轨迹
2. **安全验证**：在危险环境中由人类监督执行
3. **技能迁移**：将人类操作技能直接迁移到机器人
4. **数据多样性**：不同操作者提供多样化操作策略
5. **失败恢复**：人类可及时干预纠正错误

## §2 — ALOHA (A Low-cost Open-source Hardware System for Bimanual Teleoperation)

### 2.1 系统概述

ALOHA由斯坦福于2023年提出，是一个**低成本、开源的双臂遥操作硬件系统**。

| 参数 | 规格 |
|------|------|
| 机械臂 | 2× ViperX 300 (6DoF) 或 2× WidowX 250s |
| 控制器 | 2× 同型号机械臂（主臂） |
| 夹爪 | 2× 平行夹爪 |
| 相机 | 4× RGB (顶部、左右腕部、正面) |
| 计算单元 | 标准PC |
| 总成本 | ~$20,000 |
| 开源 | 是（硬件+软件） |

### 2.2 硬件架构

```
┌─────────────────────────────────────────────────────┐
│                  ALOHA 系统架构                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│   操作者                                            │
│     │                                               │
│     ▼                                               │
│   ┌─────────────┐                                   │
│   │  主臂 (Leader) │  <-- 人手持                   │
│   │  2× ViperX    │      直接操控                  │
│   └──────┬──────┘                                   │
│          │                                          │
│          │ 位置指令                                  │
│          ▼                                          │
│   ┌─────────────┐                                   │
│   │  从臂 (Follower)│  <-- 执行任务                │
│   │  2× ViperX    │                               │
│   └──────┬──────┘                                   │
│          │                                          │
│          ▼                                          │
│   ┌─────────────┐                                   │
│   │  相机阵列    │                                   │
│   │  4× RGB     │  <-- 记录操作过程                │
│   └─────────────┘                                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2.3 主从映射策略

ALOHA采用**位置-位置映射**（Position-Position Mapping）：

```
主臂关节角度 θ_leader ──→ 从臂关节角度 θ_follower

其中：θ_follower = θ_leader + θ_offset

θ_offset: 初始位置偏移量（可手动调整）
```

**映射特点**：
- **同构映射**：主从臂为相同型号，关节一一对应
- **直接映射**：无需逆运动学求解
- **重力补偿**：主臂有重力补偿，操作者无需费力支撑
- **缩放因子**：可选位置缩放（如主臂移动1cm，从臂移动0.5cm）

### 2.4 数据采集与ACT算法

ALOHA采集的数据格式：

```python
# ALOHA数据帧结构
data_frame = {
    "timestamp": float,           # 时间戳
    "observation": {
        "images": {
            "top": ndarray,       # 顶部相机图像 (480×640×3)
            "left_wrist": ndarray, # 左腕相机图像 (480×640×3)
            "right_wrist": ndarray,# 右腕相机图像 (480×640×3)
            "front": ndarray,     # 正面相机图像 (480×640×3)
        },
        "joint_positions": ndarray,  # 从臂关节角度 (14 DoF)
        "gripper_states": ndarray,   # 夹爪开度 (2)
    },
    "action": ndarray,             # 目标关节位置 (14 DoF + 2 gripper)
}
```

**ACT (Action Chunking with Transformers)**：
- 基于Transformer的策略架构
- 预测未来一段时间的动作序列（chunking）
- 使用CVAE编码动作分布
- 在ALOHA数据上训练，实现双臂协调操作

### 2.5 ALOHA的局限

| 局限 | 说明 |
|------|------|
| 固定基座 | 无法处理需要移动的任务 |
| 工作空间 | 受限于从臂臂展 |
| 同构要求 | 主从臂需相同型号 |
| 成本 | 仍需$20,000，门槛较高 |
| 力反馈 | 无双向力反馈 |
| 单操作者 | 需双手同时操作双臂 |

## §3 — Mobile ALOHA

### 3.1 系统概述

Mobile ALOHA在ALOHA基础上增加了**移动基座**，扩展了工作空间。

| 参数 | 规格 |
|------|------|
| 移动基座 | AgileX Tracer (履带式) |
| 机械臂 | 2× ViperX 300 |
| 高度调节 | 电动升降柱 |
| 相机 | 4× RGB + 可选深度相机 |
| 总成本 | ~$32,000 |
| 续航 | 约2-3小时 |

### 3.2 移动操作挑战

**新增控制维度**：
- 基座移动 (X, Y, θ)
- 升降柱高度 (Z)
- 总计：基座3DoF + 升降1DoF + 双臂14DoF + 夹爪2DoF = 20 DoF

**控制方式**：
- 基座：手柄/键盘控制
- 机械臂：同ALOHA主从映射
- 协调：操作者同时控制移动和操作

### 3.3 Mobile ALOHA的数据特性

| 特性 | 说明 |
|------|------|
| 任务复杂度 | 更高（需考虑移动+操作协调） |
| 数据量 | 更大（包含移动轨迹） |
| 成功标准 | 更复杂（位置+姿态+交互） |
| 失败模式 | 更多（碰撞、定位误差等） |

## §4 — Diffusion Policy

### 4.1 核心思想

Diffusion Policy由Columbia University和MIT于2023年提出，将**扩散模型 (Diffusion Model)** 应用于机器人策略学习。

**关键洞见**：
- 传统策略：$a_t = \pi(o_t)$，确定性或高斯分布
- Diffusion Policy：通过去噪过程生成动作序列
- 优势：可建模多模态动作分布，表达复杂操作

### 4.2 Diffusion Policy架构

```
输入观测 o_t (图像 + 状态)
       │
       ▼
┌─────────────────────┐
│ 视觉编码器 (ResNet/ViT)│
│ 提取视觉特征         │
└─────────┬───────────┘
           │
           ▼
┌─────────────────────┐
│ 条件扩散模型         │
│ 生成动作序列         │
│                     │
│ 前向加噪: a^(k) = √(α_k)·a^(0) + √(1-α_k)·ε  │
│ 反向去噪: ε_θ(a^(k), o_t, k)                  │
└─────────┬───────────┘
           │
           ▼
    输出动作序列 [a_t, a_{t+1}, ..., a_{t+T}]
```

### 4.3 Diffusion Policy在遥操作中的应用

**训练数据来源**：
- ALOHA/Mobile ALOHA采集的遥操作数据
- UMI手持设备数据
- 人类操作视频（Ego-centric）

**优势**：

| 优势 | 说明 |
|------|------|
| 多模态动作 | 可表达"向左推"或"向右推"等多种选择 |
| 平滑轨迹 | 生成的动作序列自然平滑 |
| 数据效率 | 相比BC-RNN，需要更少数据 |
| 鲁棒性 | 对噪声和遮挡更鲁棒 |
| 可解释性 | 可通过可视化去噪过程理解策略 |

### 4.4 Diffusion Policy vs ACT

| 维度 | Diffusion Policy | ACT |
|------|------------------|-----|
| 模型类型 | 扩散模型 | Transformer + CVAE |
| 动作分布 | 多模态 | 单模态（均值） |
| 训练稳定性 | 高 | 中（需平衡KL散度） |
| 推理速度 | 较慢（需多步去噪） | 快（单次前向） |
| 数据需求 | 较少 | 较多 |
| 适用任务 | 复杂多解任务 | 确定性任务 |
| 代码开源 | 是 | 是 |

### 5.3 VR遥操作的优势与挑战

**优势**：
- **沉浸感**：操作者获得第一人称视角
- **直观性**：自然的手部动作映射
- **安全性**：远程操作，避免直接接触
- **可扩展性**：可控制异地机器人

**挑战**：

| 挑战 | 说明 | 解决方案 |
|------|------|----------|
| 延迟 | VR渲染+网络+机器人响应 | 预测渲染、本地控制 |
| 晕动症 | 视觉-前庭不匹配 | 高帧率(≥90fps)、低延迟(<20ms) |
| 深度感知 | 单目/立体显示深度不准 | 深度相机叠加、立体视觉 |
| 力反馈缺失 | 无触觉反馈 | 力反馈手套、振动反馈 |
| 操作精度 | 手柄抖动影响精度 | 滤波、缩放映射 |
| 长时间使用 | VR头显佩戴不适 | 轻量化设计、间歇休息 |

## §6 — 主从映射策略详解

### 6.1 映射策略分类

| 映射类型 | 描述 | 公式 | 适用场景 |
|----------|------|------|----------|
| 位置-位置 | 主臂位置→从臂位置 | θ_f = θ_l + offset | 同构机械臂 |
| 位置-速度 | 主臂位置→从臂速度 | θ̇_f = K_p(θ_l - θ_f) | 异构机械臂 |
| 速度-速度 | 主臂速度→从臂速度 | θ̇_f = θ̇_l | 精细控制 |
| 力-位置 | 主臂力→从臂位置 | 力控跟随 | 接触任务 |
| 任务空间 | 末端位姿→末端位姿 | X_f = X_l + offset | 不同构型 |
| 关节空间 | 关节角度→关节角度 | 直接映射 | 同构机械臂 |

### 6.2 缩放与偏移

**位置缩放**：
```
θ_f = K_scale × θ_l + θ_offset

K_scale: 缩放因子 (通常 0.5 - 2.0)
θ_offset: 位置偏移 (调整工作空间)
```

**速度缩放**：
```
θ̇_f = K_v × θ̇_l

K_v: 速度缩放因子
- K_v > 1: 放大操作者动作（精细操作）
- K_v < 1: 缩小操作者动作（大范围移动）
```

### 6.3 滤波与平滑

**低通滤波**：减少操作者手抖影响
```
θ_filt(t) = α × θ_raw(t) + (1-α) × θ_filt(t-1)

α: 滤波系数 (0 < α < 1)
- α 接近1: 响应快，噪声大
- α 接近0: 响应慢，更平滑
```

**死区处理**：忽略小范围输入
```
if |θ_l - θ_l(prev)| < θ_deadzone:
    θ_l = θ_l(prev)
```
