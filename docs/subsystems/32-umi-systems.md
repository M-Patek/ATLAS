# UMI手持系统

## §0 — One-liner

UMI (Universal Manipulation Interface) 是一种低成本、便携的手持数据采集系统，通过腕部相机和夹爪将人类操作直接转化为机器人可学习的策略。

## §1 — 核心概念

### 1.1 UMI的设计哲学

UMI由斯坦福大学李飞飞团队于2023年提出，其核心设计哲学包括：

1. **便携性**：无需固定机械臂，手持即可完成数据采集
2. **低成本**：整体硬件成本<$500，大幅降低数据采集门槛
3. **通用性**：适用于多种操作任务和不同机器人平台
4. **可扩展性**：开源硬件设计，易于复制和改进

### 1.2 UMI vs Ego-centric vs 遥操作

| 维度 | UMI | Ego-centric | 遥操作 |
|------|-----|-------------|--------|
| 操作方式 | 手持夹爪 | 自然手部操作 | 主从机械臂控制 |
| 视角 | 腕部向下 | 头部/胸戴 | 外部固定相机 |
| 物理交互 | 真实接触物体 | 可能无物理交互 | 通过机械臂间接交互 |
| 便携性 | 高（手持） | 高（穿戴） | 低（固定设备） |
| 成本 | $300-500 | $400-3500 | $10,000+ |
| 学习难度 | 低 | 中 | 高 |
| 数据质量 | 高（直接对应） | 中（需视角转换） | 高（精确轨迹） |
| 适用任务 | 桌面操作、抓取 | 日常活动 | 双臂协作、精密操作 |

## §2 — 手持夹爪的机械设计

### 2.1 硬件组成

UMI手持设备的核心组件：

```
┌─────────────────────────────────────┐
│           UMI手持设备                │
│                                     │
│    ┌───────────────┐               │
│    │   腕部相机     │  ← GoPro/   │
│    │  (向下拍摄)    │    类似相机   │
│    └───────┬───────┘               │
│            │                        │
│    ┌───────┴───────┐              │
│    │    夹爪机构    │              │
│    │  ┌───┐ ┌───┐  │              │
│    │  │   │ │   │  │  ← 平行夹爪   │
│    │  └───┘ └───┘  │              │
│    └───────┬───────┘              │
│            │                        │
│    ┌───────┴───────┐              │
│    │   手柄/触发器   │  ← 人手操作  │
│    └───────────────┘               │
└─────────────────────────────────────┘
```

### 2.2 夹爪设计详解

**平行夹爪机构**：
- **驱动方式**：舵机 (Servo Motor, e.g., SG90) 或线性致动器
- **行程**：约50mm（可调换不同手指）
- **夹持力**：约5-10N（足够轻物体操作）
- **手指材质**：3D打印PLA + 硅胶垫（增加摩擦）
- **开合检测**：电位器或霍尔传感器反馈开度

**关键设计要点**：

| 设计要素 | 实现方式 | 目的 |
|----------|----------|------|
| 平行夹持 | 滑轨+连杆机构 | 保持夹持面平行，稳定抓取 |
| 力反馈 | 可选FSR压力传感器 | 检测夹持力度 |
| 快换手指 | 磁吸/插销连接 | 适应不同物体形状 |
| 轻量化 | 3D打印结构件 | 减少操作者疲劳 |

### 2.3 手柄设计

**人体工学考虑**：
- **握持方式**：类枪式握持，食指控制触发器
- **触发器映射**：
  - 按下：夹爪闭合（抓取）
  - 释放：夹爪张开（释放）
  - 行程连续：映射到夹爪开度（比例控制）
- **重量分布**：重心靠近手腕，减少操作疲劳
- **扩展接口**：预留IMU、按钮、LED指示灯接口

### 2.4 开源硬件资源

UMI官方提供完整的开源硬件设计：

| 资源 | 链接/说明 |
|------|----------|
| 3D模型 | STL/STEP格式，支持FDM打印 |
| BOM清单 | 完整物料清单，含供应商链接 |
| 装配指南 | 分步图文教程 |
| 固件 | Arduino/ESP32代码 |
| 成本估算 | ~$300-500（不含相机） |

## §3 — 腕部相机配置

### 3.1 相机选型与安装

**推荐相机**：

| 相机型号 | 分辨率 | 帧率 | FOV | 重量 | 价格 |
|----------|--------|------|-----|------|------|
| GoPro Hero 10/11 | 5.3K | 60fps | ~122° | 153g | $250 |
| Insta360 GO 3 | 2.7K | 30fps | ~120° | 35g | $380 |
| DJI Action 4 | 4K | 120fps | ~155° | 145g | $400 |
| 定制USB相机 | 1080p | 60fps | ~120° | 20g | $50 |

**安装要点**：
- **视角方向**：相机光轴与夹爪开合方向平行（向下拍摄）
- **安装位置**：手腕上方或侧面，不遮挡夹爪视野
- **刚性连接**：相机与夹爪刚性固定，保持手眼标定不变
- **保护罩**：透明亚克力罩防止碰撞损坏镜头

### 3.2 视角与成像特性

**典型UMI视角**：

```
        相机视野 (FOV ~120°)
              │
              ▼
    ┌─────────────────────┐
    │                     │
    │    工作台面/桌面      │
    │         ┌───┐       │
    │         │物体│       │
    │         └───┘       │
    │    ┌───────────┐    │
    │    │   夹爪     │    │
    │    │  ┌─┐ ┌─┐  │    │
    │    │  └─┘ └─┘  │    │
    │    └───────────┘    │
    └─────────────────────┘
         (第一人称视角)
```

**关键成像参数**：
- **工作距离**：夹爪指尖到物体约10-30cm
- **景深要求**：覆盖操作区域（约20cm深度范围）
- **分辨率需求**：物体在图像中至少占100×100像素
- **帧率要求**：≥30fps以捕捉快速操作

### 3.3 手眼标定

UMI系统的核心挑战之一：**相机坐标系与夹爪坐标系的精确标定**。

**标定方法**：
1. **棋盘格标定法**：
   - 在夹爪上固定已知尺寸的棋盘格
   - 相机从不同角度拍摄
   - 解算相机到夹爪的变换矩阵 $T_{cam}^{gripper}$

2. **ArUco标记法**：
   - 在夹爪手指上粘贴ArUco标记
   - 实时检测标记位姿
   - 在线更新手眼变换

3. **自标定（Self-calibration）**：
   - 利用已知几何约束（如夹爪闭合时指尖距离）
   - 通过多次操作迭代优化标定参数

**标定精度要求**：
- 位置误差 < 2mm
- 角度误差 < 1°
- 定期重新标定（温度变化、机械磨损）

## §4 — UMI数据采集流程

### 4.1 标准采集流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  步骤1:准备  │ -> │  步骤2:采集  │ -> │  步骤3:后处理 │
└─────────────┘    └─────────────┘    └─────────────┘
      │                  │                  │
      ▼                  ▼                  ▼
  检查设备          执行操作任务         视频分割
  标定手眼          同步记录           动作标注
  设置场景          多视角采集         数据对齐
  预演动作          质量检查           格式转换
```

### 4.2 采集协议

**场景设置**：
1. **工作台面**：平整、颜色均匀（推荐白色/灰色）
2. **照明**：顶部均匀照明，避免阴影
3. **背景**：简洁，减少无关物体干扰
4. **物体摆放**：随机位置，覆盖不同姿态

**操作执行**：
1. **任务定义**：明确操作目标（如"将杯子从A移到B"）
2. **演示次数**：每个任务至少10-20次演示
3. **变体采集**：不同起始位置、不同物体、不同速度
4. **失败案例**：故意采集部分失败案例（用于学习恢复策略）

### 4.3 数据记录内容

| 数据类型 | 来源 | 频率 | 格式 |
|----------|------|------|------|
| RGB视频 | 腕部相机 | 30-60fps | MP4/H.264 |
| 夹爪开度 | 编码器/电位器 | 30-60Hz | CSV |
| 夹爪力 | FSR传感器（可选） | 30-60Hz | CSV |
| 6DoF位姿 | 外置追踪（可选） | 30-120Hz | CSV/JSON |
| 时间戳 | 系统时钟 | - | 统一基准 |

### 4.4 数据标注

**自动标注**：
- **夹爪状态**：开/合/部分闭合（基于编码器阈值）
- **动作分段**：基于夹爪状态变化自动分段
- **物体检测**：使用预训练检测器自动标注

**人工标注**：
- **动作类别**：抓取、放置、推拉、旋转等
- **成功/失败**：操作是否成功完成
- **注释**：特殊情况说明

## §5 — 与Ego-centric采集的对比

### 5.1 数据特性对比

| 特性 | UMI | Ego-centric (Vision Pro) |
|------|-----|--------------------------|
| 视角稳定性 | 高（手部支撑） | 中（头部晃动） |
| 操作精度 | 高（直接物理交互） | 中（隔空操作） |
| 手部可见性 | 始终完整可见 | 可能自遮挡 |
| 深度信息 | 双目/结构光可选配 | LiDAR内置 |
| 操作自然度 | 中（手持设备） | 高（自然手势） |
| 物理约束 | 真实物理反馈 | 无（虚拟交互） |
| 数据采集效率 | 高（单次操作即数据） | 中（需后期处理） |
| 机器人迁移性 | 高（直接对应） | 中（需视角转换） |

### 5.2 适用任务对比

| 任务类型 | UMI | Ego-centric |
|----------|-----|-------------|
| 桌面抓取 | ✅✅✅ | ✅✅ |
| 抽屉开关 | ✅✅✅ | ✅✅ |
| 按钮按压 | ✅✅✅ | ✅✅ |
| 工具使用 | ✅✅ | ✅ |
| 烹饪操作 | ⚠️ | ✅✅✅ |
| 移动操作 | ⚠️ | ✅✅✅ |
| 双臂协作 | ❌ | ✅ |
| 社交交互 | ❌ | ✅✅✅ |

## §6 — 与DVAS项目的关联

在DVAS项目中，UMI系统作为**精细化操作数据采集的核心工具**：

### 6.1 在DVAS Pipeline中的位置

```
┌─────────────────────────────────────────┐
│           DVAS 数据采集层                │
├─────────────────────────────────────────┤
│  Ego-centric ──→ 日常活动、移动操作      │
│  UMI ──────────→ 桌面精细操作          │
│  Sim2Real ────→ 大规模预训练           │
│  遥操作 ───────→ 双臂/精密任务          │
└─────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│  UMI数据 → 动作策略学习 → 夹爪控制      │
│           (Diffusion Policy/ACT)        │
└─────────────────────────────────────────┘
```

### 6.2 DVAS推荐的UMI配置

**基础配置**（~$400）：
- 3D打印夹爪机构（PLA）
- SG90舵机 × 1
- GoPro Hero 10（二手）
- Arduino Nano + 扩展板
- 基础手柄

**增强配置**（~$800）：
- 碳纤维增强夹爪
- 数字舵机（更高精度）
- Insta360 GO 3（更轻便）
- ESP32-S3（WiFi/蓝牙传输）
- FSR力传感器 × 2
- 6DoF IMU

**研究配置**（~$1500）：
- 精密线性致动器
- RealSense D405（腕部深度相机）
- OptiTrack/Vicon外部位姿追踪
- 力/扭矩传感器

### 6.3 UMI数据在DVAS中的应用

1. **策略学习**：UMI采集的轨迹直接用于训练Diffusion Policy或ACT
2. **数据增强**：结合Sim2Real在仿真中生成变体数据
3. **跨平台迁移**：UMI数据可迁移到不同机器人（Franka, xArm等）
4. **评估基准**：建立标准化的UMI操作任务评估集
